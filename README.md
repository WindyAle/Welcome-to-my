# HouSKetch

**핵심: 주관적인 텍스트 → 객관적인 평가**

LLM의 개입 위치: 게임의 처음(의뢰)과 끝(평가)

### 1. 고객 의뢰 생성 (LLM Chat)

`ollama`의 채팅 모델(Llama 3 등)을 '고객 페르소나 생성기'로 사용

- **시스템 프롬프트 (노출되지 않음):** "당신은 곧 이사할 집의 인테리어를 의뢰하는 고객입니다. 당신의 직업, 성격, 예산을 포함하여 독특하고 구체적인 요구사항을 1~2문장으로 설명하세요."
- **LLM 생성 (예시 1: 미니멀리스트):**
    
    > "저는 명상을 즐기는 요가 강사입니다. 흰색과 베이지색 톤의 깔끔하고 정돈된 공간을 원해요. 불필요한 가구는 모두 치워주세요. '젠(Zen)' 스타일이 좋습니다."
    > 
- **LLM 생성 (예시 2: 수집가):**
    
    > "저는 빈티지 장난감 수집가입니다. 어두운 목재(다크 아카데미아) 풍의 서재가 필요해요. 수집품을 놓을 선반이 아주 많아야 하고, 벽난로 앞에 붉은색의 푹신한 소파가 있었으면 좋겠어요."
    > 
    

### 2. 고객 만족도 평가 (한국어 모델 `EEVE` 활용)

고객의 평점은 '정답'이 아닌 '유사도'로 매겨져야 한다.

1. **[A] 의뢰서 벡터화:**
    - 고객이 `"젠 스타일의 깔끔한 공간"`이라는 의뢰서를 제출
    - 이 문장을 `ollama EEVE` 모델에 보내 '의미 벡터' **A**를 받아 저장
2. **[B] 플레이어 디자인 벡터화:**
    - 플레이어가 디자인을 완료
    - 게임 엔진은 플레이어가 배치한 모든 가구 아이템의 목록을 텍스트로 추출합니다.
    - `"흰색 소파, 유리 테이블, 식물 2개, 카펫"`
    - 이 '디자인 설명서'를 `EEVE` 모델에 보내 '의미 벡터' **B**를 추출
3. **[Score] 코사인 유사도 계산:**
    - [A] 벡터와 [B] 벡터의 코사인 유사도를 계산
    - **유사도 95%:** 5점 만점. "완벽해요! 제가 찾던 바로 그 '젠' 스타일입니다!"
    - **유사도 60%:** 3점. "음... 제가 원하던 느낌과는 조금 다르네요."
    - **유사도 10%:** 1점. "환불은 어떻게 하죠? 최악이네요."
4. **[Feedback] 상세 피드백 생성 (LLM Chat 모델 활용):**
    - 단순히 점수만 주는 것이 아니라, **왜** 그 점수가 나왔는지 LLM(채팅 모델)이 설명해준다.
    - **프롬프트:** "고객은 [A: 젠 스타일]을 원했습니다. 디자이너는 [B: 흰색 소파, 유리 테이블]을 디자인했습니다. [Score: 4.5/5]가 나왔습니다. 고객의 입장에서 상세한 피드백을 작성해 주세요."
    - **LLM 응답:** "거의 완벽해요! 흰색 소파와 유리 테이블이 공간을 정말 넓고 평화롭게 만드네요. 다만 식물이 하나쯤 더 있었으면... 그래도 4.5점 드립니다!"

# 구현

### 그리드 기반 맵

- 플레이어에게 2D 격자(Grid)로 된 빈 방이 주어진다.
- 플레이어는 UI에서 '소파(2x1)', '침대(2x2)', '테이블(1x1)' 같은 2D 스프라이트(이미지)를 드래그 앤 드롭으로 배치한다.
- 3D는 요구사항이 너무 높음 → 2D 탑다운 뷰 방식
